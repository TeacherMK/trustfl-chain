# Public Release Checklist

This repository is intended to be released as a public source-code and reproducibility artifact for TrustFL-Chain. The code supports a simulator-level claim: lightweight, auditable, reputation-aware poisoned-update defense for Non-IID federated learning.

## Scope to State Publicly

- TrustFL-Chain is a PyTorch research prototype, not a production blockchain system.
- The local hash-chain ledger provides compact auditability and reproducibility support.
- The implementation does not provide decentralized consensus, secure aggregation, smart contracts, trusted execution, or zero-knowledge proofs.
- Fashion-MNIST is the main multi-seed evidence. CIFAR-10 is supplemental validation.

## Files Suitable for Release

- `src/`: simulator, attacks, defenses, aggregation, ledger, and utilities.
- `scripts/`: experiment runners, aggregation scripts, and figure generation scripts.
- `figures/final_artifacts/`: generated study figures.
- `results/final_artifacts/`: aggregate CSVs and statistical summaries.
- `results/comparison_summary/`: compact summary for the direction-filter comparison.

## Files to Exclude or Regenerate

- `.venv/`
- `data/`
- `__pycache__/`
- `*.pyc`
- local private notes and temporary build files
- very large raw per-run outputs unless the target artifact platform allows them

## Reproducibility Commands

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python -m compileall src scripts
python scripts/run_final_artifacts_experiments.py --manifest-only
python scripts/aggregate_final_artifacts_results.py
python scripts/plot_final_artifacts_figures.py
```

## Final Pre-Upload Checks

- Confirm `results/final_artifacts/missing_or_failed_runs.csv` is empty or explained.
- Confirm `results/final_artifacts/aggregate_mean_std.csv` matches the numbers reported in the study summary.
- Confirm all public figures in `figures/final_artifacts/` can be regenerated by the plotting scripts.
- Confirm no private absolute paths appear in README examples, release metadata, or public reports.
- Confirm the README keeps the claims cautious and does not describe the ledger as a production blockchain.
- Confirm local private notes are not included in the public repository.
